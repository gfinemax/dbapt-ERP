# 회계/자금 지출결의서 관리 사용자 매뉴얼

작성일: 2026-07-03

## 1. 화면 위치

지출결의서 관리는 조합 사무국의 지출을 결재, 지급처리, 전표생성까지 이어서 관리하는 화면이다.

```text
회계/자금 > 전표·증빙관리 > 지출결의서 관리
```

이 화면에서는 지출전표를 바로 입력하지 않고, 먼저 지출결의서를 작성한 뒤 내부 결재를 완료하고 실제 지급처리를 기록한다. 지급이 끝난 뒤에는 지출전표를 생성하여 회계 전표와 증빙자료를 연결한다.

## 2. 기본 업무 흐름

```text
지출결의서 작성
→ 승인요청
→ 장현제 부장 결재
→ 오학동 사무장 결재
→ 안동연 조합장 최종 결재
→ 승인완료
→ 지급대기
→ 지급처리
→ 지급완료
→ 지출전표 생성
→ 증빙자료 연결
→ 처리이력 기록
```

결재는 내부 승인 절차이며, 실제 계좌이체나 카드결제는 지급처리 단계에서 별도로 기록한다.

## 3. 결의서 유형

### 단일 지출결의

하나의 거래처, 하나의 지출 건을 결의할 때 사용한다.

예시:

- 사무실 임대료 지급
- 법무비 지급
- 세무자문료 지급
- 감정평가비 지급
- 조합원 환불금 지급

### 프로젝트 일괄 지출결의

하나의 사업과제나 프로젝트 안에 여러 지출 항목이 있을 때 사용한다. 결재는 결의서 단위로 한 번 진행하고, 지급처리와 전표생성은 일괄 또는 항목별로 처리할 수 있다.

예시:

- 정기총회 준비 비용
- 이사회 운영 비용
- 조합원 안내문 발송 비용
- 홈페이지·ERP 구축 비용
- 토지매입 관련 복수 비용

## 4. 지출결의서 작성

상단의 `지출결의 작성` 버튼을 클릭하면 작성 모달이 열린다.

입력 순서:

1. `결의서 유형`을 선택한다.
2. 단일 지출결의는 지출구분, 예산항목, 거래처명, 지급정보, 금액정보를 입력한다.
3. 프로젝트 일괄 지출결의는 프로젝트/사업과제를 선택하고 세부 지출내역을 행 단위로 입력한다.
4. 지출사유, 관련계약, 관련회의/의결, 증빙자료, 내부메모를 입력한다.
5. `임시저장` 또는 `승인요청`을 클릭한다.

`임시저장`을 누르면 승인상태는 `작성중`, 지급상태는 `지급전`으로 저장된다.

`승인요청`을 누르면 승인상태는 `승인대기`가 되고 현재결재자는 `장현제 부장`으로 지정된다.

## 5. 프로젝트 일괄 지출결의 작성

프로젝트 일괄 지출결의에서는 세부 지출내역을 여러 행으로 입력한다.

세부 행 주요 항목:

- 지출예정일
- 거래처
- 계정항목
- 지출구분
- 예산항목
- 지출항목명
- 내역 및 산출근거
- 공급가액
- 부가세
- 증빙유형
- 증빙파일명
- 지급방법
- 지급상태

총지급액은 세부 행의 공급가액과 부가세 합계로 자동 계산된다.

## 6. 일괄 결의 지급 방식

프로젝트 일괄 지출결의는 지급처리 방식을 선택할 수 있다.

### 일괄 지급

모든 세부 항목을 한 번에 지급완료 처리한다.

사용 상황:

- 동일한 날짜에 전체 항목을 한 번에 지급하는 경우
- 하나의 프로젝트 비용을 한 번에 집행하는 경우

처리 결과:

- 결의서 지급상태가 `지급완료`로 변경된다.
- 세부 항목 지급상태도 모두 `지급완료`로 변경된다.
- 처리이력에 지급완료 이력이 기록된다.

### 항목별 지급

세부 지출내역의 각 행을 따로 지급완료 처리한다.

사용 상황:

- 거래처별 지급일이 다른 경우
- 일부 항목만 먼저 지급하는 경우
- 세부 항목별 이체확인증을 따로 관리해야 하는 경우

처리 결과:

- 일부 항목만 지급되면 결의서 지급상태가 `부분지급`으로 변경된다.
- 모든 항목이 지급되면 결의서 지급상태가 `지급완료`로 변경된다.
- 각 지급 처리마다 처리이력에 지급완료 또는 부분지급 이력이 기록된다.

## 7. 결재함

`결재함` 탭에는 현재 사용자가 결재해야 할 문서만 표시된다.

현재 사용자 mock 기준:

```text
오학동 사무장
```

따라서 현재결재자가 `오학동 사무장`인 지출결의서만 결재함에 표시된다.

결재 순서:

```text
장현제 부장 → 오학동 사무장 → 안동연 조합장
```

승인 버튼을 누르면 현재 결재자가 승인완료되고 다음 결재자로 넘어간다. 안동연 조합장까지 승인하면 승인상태는 `승인완료`, 지급상태는 `지급대기`가 된다.

## 8. 반려 처리

목록 또는 상세 화면에서 `반려` 버튼을 클릭하면 반려사유 입력 모달이 열린다.

반려사유는 필수 입력값이다.

반려 처리 결과:

- 승인상태가 `반려`로 변경된다.
- 지급상태가 `지급전`으로 변경된다.
- 현재결재자가 비워진다.
- 반려사유가 목록과 상세 화면에 표시된다.
- 처리이력에 반려 이력이 기록된다.

## 9. 지급대기와 부분지급

최종 결재가 완료된 문서는 `지급대기` 탭에 표시된다.

프로젝트 일괄 지출결의에서 일부 항목만 지급한 경우에는 `부분지급` 탭에 표시된다.

지급처리 모달 입력 항목:

- 지급일
- 지급방법
- 실제지급액
- 지급은행
- 지급계좌
- 예금주
- 이체확인증 첨부
- 지급메모

지급처리는 실제 은행 이체를 실행하는 기능이 아니라, 사무국에서 실제 지급한 결과를 ERP에 기록하는 기능이다.

## 10. 지급완료

지급처리가 완료된 문서는 `지급완료` 탭에 표시된다.

지급완료 상태에서는 지출전표를 생성할 수 있다.

지급완료 후 확인할 항목:

- 지급일
- 지급방법
- 실제지급액
- 지급은행
- 지급계좌
- 예금주
- 이체확인증 첨부 여부
- 지급메모
- 처리이력

## 11. 지출전표 생성

지급완료 상태의 결의서에서 `지출전표 생성` 버튼을 클릭하면 지출전표 mock 데이터가 생성된다.

### 통합 전표

하나의 결의서에 하나의 전표번호를 생성한다.

예시:

```text
VOU-2026-0001
```

### 항목별 전표

프로젝트 일괄 지출결의에서 세부 항목별로 전표번호를 생성한다.

예시:

```text
VOU-2026-0001-01
VOU-2026-0001-02
```

여러 거래처 또는 여러 계정항목이 섞여 있는 일괄 결의서는 항목별 전표 생성을 사용하는 것이 좋다.

## 12. 상세보기

목록에서 `상세보기` 버튼을 클릭하면 지출결의서 상세 화면이 열린다.

상세 화면에서 확인할 수 있는 항목:

- 기본정보
- 지출정보
- 지급정보
- 예산반영
- 연결정보
- 결재선
- 증빙자료
- 처리이력
- 출력보관

프로젝트 일괄 지출결의는 상단에 프로젝트 요약과 세부 지출내역 표가 추가로 표시된다.

## 13. 처리이력

처리이력은 감사 대응을 위해 결의서의 주요 상태 변경을 시간순으로 기록한다.

기록되는 주요 이력:

- 지출결의서 작성
- 임시저장
- 승인요청
- 결재승인
- 반려
- 지급대기 전환
- 부분지급
- 지급완료
- 지급보류
- 전표 생성
- 증빙자료 첨부
- 출력 미리보기
- 보관용 PDF 생성

상세 화면의 처리이력 영역에서 `오래된순` 또는 `최신순`으로 정렬할 수 있다.

## 14. 출력 및 보관

상세 화면에서 `출력 미리보기` 또는 `보관용 PDF 생성` 버튼을 사용할 수 있다.

보관용 출력 전 확인하는 항목:

- 거래처 입력 여부
- 지출사유 입력 여부
- 총지급액 입력 여부
- 예산항목 선택 여부
- 예산초과 사유 입력 여부
- 결재선 존재 여부

보관용 PDF 생성 시 출력번호와 보관위치가 기록되고, 처리이력에도 보관용 PDF 생성 이력이 남는다.

## 15. 탭 기준

| 탭 | 표시 기준 |
| --- | --- |
| 전체 | 모든 지출결의서 |
| 내가 작성한 결의서 | 작성자가 현재 사용자와 같은 결의서 |
| 결재함 | 현재결재자가 현재 사용자이고 승인상태가 승인대기인 결의서 |
| 반려함 | 승인상태가 반려인 결의서 |
| 지급대기 | 승인상태가 승인완료이고 지급상태가 지급대기인 결의서 |
| 부분지급 | 지급상태가 부분지급인 결의서 |
| 지급완료 | 지급상태가 지급완료인 결의서 |
| 보류 | 지급상태가 보류인 결의서 |
| 전표생성완료 | 전표번호가 생성된 결의서 |

## 16. 상태값

### 승인상태

| 상태 | 의미 |
| --- | --- |
| 작성중 | 임시저장 상태 |
| 승인대기 | 결재자 승인이 필요한 상태 |
| 승인완료 | 최종 결재까지 완료된 상태 |
| 반려 | 결재자가 반려한 상태 |

### 지급상태

| 상태 | 의미 |
| --- | --- |
| 지급전 | 아직 지급처리 전 상태 |
| 지급대기 | 최종 결재 후 실제 지급이 필요한 상태 |
| 부분지급 | 일괄 결의 중 일부 항목만 지급완료된 상태 |
| 지급완료 | 실제 지급 결과가 기록된 상태 |
| 보류 | 지급이 보류된 상태 |

## 17. 운영비 처리 팁

운영비 지출결의는 예산항목을 반드시 확인한 뒤 작성한다.

예시:

- 운영비 > 임대료
- 운영비 > 사무용품비
- 운영비 > 소모품비
- 운영비 > 통신비
- 운영비 > 지급수수료
- 운영비 > 예비비

예산을 초과하는 경우에는 `예산초과 사유`를 입력해 보관용 출력 시 감사 대응 근거를 남기는 것이 좋다.

선지급한 운영비는 지급유형을 `선지급` 또는 `사후정산`으로 선택하고, 선지급일, 선지급자, 선지급 금액, 실제 사용금액, 사후승인 사유를 입력한다.
