# Daebang ERP Screen Wireframes

작성일: 2026-06-06

## 목적

이 문서는 `docs/design/erp-design-concept.md`를 실제 화면 구조로 바꾼 1차 와이어프레임이다. 목표는 구현 전에 ERP의 첫 화면, 주요 모듈, 연동 모듈의 배치를 정하고, 이후 API 설계와 프론트엔드 구현이 같은 화면 목표를 바라보게 만드는 것이다.

1차 범위는 "ERP Core + 읽기 중심 연동 대시보드"다. 기존 앱을 한 번에 합치지 않고, ERP가 각 모듈의 현재 상태를 보여주며 주요 업무로 이동하는 허브 역할을 한다.

## 공통 앱 쉘

ERP 내부 화면은 좌측 내비게이션과 상단 바를 기본으로 한다.

```text
+--------------------------------------------------------------------------------+
| Top Bar: 사업지 선택 | 전역 검색 | 알림 | 사용자 메뉴                          |
+----------------------+---------------------------------------------------------+
| Sidebar              | Page Header                                             |
| - 대시보드           | 제목 / 설명 / 주요 액션                                 |
| - 조합원관리         +---------------------------------------------------------+
| - 회계/자금          | Content Area                                            |
| - 총회관리           |                                                         |
| - 토지관리           |                                                         |
| - 수지분석           |                                                         |
| - 문서/공지          |                                                         |
| - 연동상태           |                                                         |
| - 설정               |                                                         |
+----------------------+---------------------------------------------------------+
```

### 공통 규칙

- 좌측 사이드바 폭: 248px
- 상단 바 높이: 60px
- 콘텐츠 여백: 24px desktop, 16px mobile
- 페이지 제목 영역: 제목, 보조 설명, 우측 주요 액션
- 표 페이지: 검색, 필터, 상태 탭, 엑셀 내보내기 기본 제공
- 상세 정보: 우측 드로어 또는 상세 페이지
- 모든 화면 하단 또는 우측에는 데이터 출처와 마지막 동기화 시각을 표시

## 1. ERP 대시보드

대시보드는 조합 사무국이 오늘 확인해야 할 상태를 한 화면에서 보는 관제 화면이다.

```text
+--------------------------------------------------------------------------------+
| 대방동 지역주택조합 ERP                                      [보고서 내보내기] |
| 조합원, 회계/자금, 총회, 토지, 수지분석 현황을 한 화면에서 확인합니다.          |
+--------------------------------------------------------------------------------+
| [조합원 1,248명] [납부율 82.4%] [예산 집행률 64.8%] [가용 자금 42.6억]        |
| [토지 확보율 71.2%] [총회 D-12] [수지 최신]                                    |
+--------------------------------------+-----------------------------------------+
| 주의 필요                            | 연동 상태                                |
| - 미납 38건                          | VoteCast     정상   5분 전               |
| - 서류 미비 12건                     | peopleON     정상   3분 전               |
| - 미승인 지출 6건                    | ERP Core     정상   방금 전              |
| - 토지 협의 지연 4필지               | db-landon    지연   2시간 전             |
| - 총회 위임장 부족                   | valueON      정상   1일 전               |
+--------------------------------------+-----------------------------------------+
| 오늘의 업무                           | 최근 변경 이력                           |
| [조합원 등록 검토] [미납 안내 발송]  | 15:20 조합원 정보 수정                   |
| [지출 승인 검토] [토지 협의]         | 14:10 납부 증빙 등록                     |
+--------------------------------------+-----------------------------------------+
| 모듈 요약                                                                      |
| [조합원관리] [회계/자금] [총회관리] [토지관리] [수지분석] [문서/공지]           |
+--------------------------------------------------------------------------------+
```

### 주요 구성

- KPI 카드: 조합원 수, 납부율, 예산 집행률, 가용 자금, 미납 건수, 토지 확보율, 총회 예정, 수지분석 최신 기준일
- 주의 필요: 사무국이 바로 처리해야 할 위험 항목
- 연동 상태: 5개 외부 앱 연결 상태와 마지막 동기화 시각
- 오늘의 업무: 빠른 액션 버튼
- 최근 변경 이력: 감사 로그의 요약 버전
- 모듈 요약: 각 앱/모듈로 진입하는 카드

### 1차 액션

- 조합원 검색
- 미납자 보기
- 회계/자금 집행 현황 보기
- 총회 현황 보기
- 토지 확보 현황 보기
- 수지분석 최신 결과 보기
- 연동 오류 확인

## 2. 조합원 목록

조합원관리는 1차 MVP의 핵심 화면이다. peopleON 연동 데이터와 ERP 공통 조합원 ID를 연결하는 기준 화면이 된다.

```text
+--------------------------------------------------------------------------------+
| 조합원관리                                               [조합원 추가] [내보내기] |
| 조합원 원장, 계약 상태, 납부 상태, 서류 상태를 관리합니다.                      |
+--------------------------------------------------------------------------------+
| 검색: 이름/연락처/조합원번호                                                     |
| [전체] [정상] [검토중] [미납] [서류미비] [탈퇴/승계]                            |
| 필터: 동/호수 | 계약상태 | 납부상태 | 서류상태 | 연동상태                       |
+--------------------------------------------------------------------------------+
| 체크 | 조합원번호 | 이름 | 연락처 | 동/호수 | 계약 | 납부 | 서류 | 연동 | 작업 |
| ---- | ---------- | ---- | ------ | ------ | ---- | ---- | ---- | ---- | ---- |
| []   | M-000124   | 김OO | 010... | 101-1  | 정상 | 완료 | 완료 | 정상 | 보기 |
| []   | M-000125   | 박OO | 010... | 102-3  | 정상 | 미납 | 미비 | 정상 | 보기 |
| []   | M-000126   | 이OO | 010... | 배정전 | 검토 | 완료 | 검토 | 충돌 | 보기 |
+--------------------------------------------------------------------------------+
```

### 주요 구성

- 검색: 이름, 연락처, 조합원번호, 동/호수
- 상태 탭: 전체, 정상, 검토중, 미납, 서류미비, 탈퇴/승계
- 필터: 계약 상태, 납부 상태, 서류 상태, 연동 상태
- 테이블: 업무 스캔을 위한 고밀도 구조
- 일괄 작업: 문자 발송, 상태 변경, 엑셀 내보내기

### 상세 진입

행 클릭 또는 "보기" 버튼으로 조합원 상세에 들어간다. 빠른 확인은 우측 드로어, 깊은 수정은 상세 페이지를 사용한다.

```text
+--------------------------------------------------------------------------------+
| 김OO / M-000124                                      [수정] [문서 추가] [상담] |
| 정상 조합원 | 납부완료 | 서류완료 | peopleON 연동 정상                         |

| 기본 정보                                | 요약 카드                                |

|                                          | 서류상태: 완료                           |
+------------------------------------------+------------------------------------------+
```

### 탭 구성

- 기본 정보: 인적 사항, 연락처, 주소, 조합원번호
- 계약/권리: 계약 상태, 동/호수, 권리 관계, 승계/탈퇴
- 납부: 계약금, 중도금, 추가분담금, 연체, 환불
- 문서: 계약서, 증빙, 조합원 제출 서류
- 상담/민원: 문의 이력, 처리 담당자, 상태
- 변경 이력: 누가 언제 무엇을 바꿨는지

## 4. 회계/자금 요약

회계/자금은 조합 운영 자금과 사업비 집행을 관리하는 재무 통제 화면이다.

```text
+--------------------------------------------------------------------------------+
| 회계/자금 관리                                         [지출 등록] [내보내기] |
| 입출금, 계정과목, 예산 대비 집행, 신탁/대행/시공 지출을 관리합니다.             |
+--------------------------------------------------------------------------------+
| [가용 자금 42.6억] [예산 집행률 64.8%] [미승인 지출 6건] [증빙 미첨부 4건]     |
+--------------------------------------------------------------------------------+
| 자금 흐름                                                                        |
| 구분 | 이번 달 | 누계 | 전월 대비 | 비고                                           |
| 수입 | 8.4억   | 126.2억 | +2.1억 | 조합원 분담금, 이자수익                       |
| 지출 | 5.7억   | 82.1억  | -0.4억 | 토지비, 용역비, 대행비                         |
+--------------------------------------------------------------------------------+
| 예산 대비 집행                                                                    |
| 계정과목 | 예산 | 집행 | 잔여 | 집행률 | 상태                                      |
| 토지비   | ...  | ...  | ...  | 72%    | 정상                                      |
| 업무대행 | ...  | ...  | ...  | 61%    | 검토                                      |
| 시공관련 | ...  | ...  | ...  | 18%    | 예정                                      |
+--------------------------------------------------------------------------------+
| 지출 승인 대기                                                                    |
| 요청일 | 거래처 | 계정과목 | 금액 | 증빙 | 상태 | 담당 | 작업                       |
+--------------------------------------------------------------------------------+
```

### 핵심 UX

- 조합원 납부 현황과 회계 입금 원장을 연결하되, 화면은 분리한다.
- 계정과목, 거래처, 증빙, 승인 상태를 한 줄에서 확인한다.
- 신탁사, 업무대행사, 시공사, 설계/감리 등 주요 지출처를 분류한다.
- 금액은 우측 정렬하고, 확정/승인 변경은 감사 로그에 남긴다.

## 5. 총회관리 요약

VoteCast는 총회 전문 모듈로 유지하고, ERP는 요약과 결과 보존 화면을 제공한다.

```text
+--------------------------------------------------------------------------------+
| 총회관리                                                   [VoteCast 열기]      |
| 출석, 위임, 서면결의, 투표 결과를 ERP에서 요약 확인합니다.                     |
+--------------------------------------------------------------------------------+
| 예정 총회: 2026 정기총회 | D-12 | 장소: 협의중 | 상태: 준비중                    |
+--------------------------------------------------------------------------------+
| [출석 대상 1,248명] [위임장 342건] [서면결의 128건] [성원 충족 예상: 가능]     |
+--------------------------------------------------------------------------------+
| 안건별 상태                                                                     |
| 안건명                 | 유형       | 투표상태 | 결과 | 확정시각 | 원본        |
| 1. 예산 승인           | 일반의결   | 준비중   | -    | -        | VoteCast    |
| 2. 시공사 선정         | 특별의결   | 준비중   | -    | -        | VoteCast    |
+--------------------------------------------------------------------------------+
| 최근 총회 결과                                                                   |
| 총회명 | 일자 | 출석률 | 안건 수 | 가결 | 부결 | 상세                         |
+--------------------------------------------------------------------------------+
```

### 핵심 UX

- ERP에서는 총회 현황을 읽고, 확정 결과를 보존한다.
- 실제 입장 처리, 송출, 실시간 투표는 VoteCast로 이동한다.
- 확정된 의결 결과는 ERP 감사 로그와 문서 관리에 연결한다.
- 출석 대상자는 조합원 원장과 매핑되어야 한다.

## 6. 토지관리 요약

db-landon은 필지/소유자 전문 모듈로 유지하고, ERP는 확보율과 위험 필지를 요약한다.

```text
+--------------------------------------------------------------------------------+
| 토지관리                                                  [db-landon 열기]      |
| 필지, 소유자, 계약 상태, 확보율을 확인합니다.                                  |
+--------------------------------------------------------------------------------+
| [전체 필지 186] [확보 132] [협의중 34] [미계약 20] [확보율 71.2%]              |
+--------------------------------------------------------------------------------+
| 확보율 흐름                                                                     |
| 월별 확보율 그래프 또는 간단한 진행 바                                          |
+--------------------------------------------------------------------------------+
| 위험 필지                                                                       |
| 필지번호 | 소유자 | 면적 | 상태 | 이슈 | 담당 | 최근 업데이트                  |
| 000-00   | 김OO   | 120  | 협의 | 연락 지연 | 홍OO | 2일 전                       |
+--------------------------------------------------------------------------------+
| 조합원/소유자 매핑 필요                                                         |
| 소유자명 | 후보 조합원 | 신뢰도 | 작업                                          |
+--------------------------------------------------------------------------------+
```

### 핵심 UX

- 토지 확보율은 대시보드 KPI와 연결된다.
- 필지 상세와 지도 기반 작업은 db-landon에서 수행한다.
- ERP는 위험 필지, 확보율, 조합원/소유자 매핑 이슈를 보여준다.

## 7. 수지분석 요약

valueON은 사업성 분석 전문 모듈로 유지하고, ERP는 최신 결과와 시나리오 비교를 보여준다.

```text
+--------------------------------------------------------------------------------+
| 수지분석                                                  [valueON 열기]        |
| 사업성 시나리오와 최신 추정 결과를 확인합니다.                                  |
+--------------------------------------------------------------------------------+
| 최신 분석: 2026-06-06 기준 | 작성자: 홍OO | 상태: 검토중                       |
+--------------------------------------------------------------------------------+
| [총사업비] [분양수입] [추정 분담금] [손익] [민감도 위험]                       |
+--------------------------------------------------------------------------------+
| 시나리오 비교                                                                   |
| 시나리오 | 토지비 | 공사비 | 금융비 | 조합원 분담금 | 손익 | 상태             |
| 기본안   | ...    | ...    | ...    | ...           | ...  | 기준             |
| 보수안   | ...    | ...    | ...    | ...           | ...  | 검토             |
+--------------------------------------------------------------------------------+
| 가정값 변경 이력                                                                 |
| 항목 | 이전값 | 변경값 | 변경자 | 변경시각                                      |
+--------------------------------------------------------------------------------+
```

### 핵심 UX

- 금액은 우측 정렬한다.
- 모든 분석값에는 기준일과 작성자를 표시한다.
- 확정값과 시뮬레이션값을 시각적으로 구분한다.
- valueON이 현재 브라우저 직접 Supabase 사용 구조라면, ERP 연동 전 서버 API 보강이 필요하다.

## 8. 문서/공지 관리

문서/공지는 조합원 포털과 연결될 수 있는 콘텐츠 관리 화면이다.

```text
+--------------------------------------------------------------------------------+
| 문서/공지 관리                                           [공지 작성] [문서 추가] |
| 조합원에게 공개할 공지와 내부 문서를 관리합니다.                                |
+--------------------------------------------------------------------------------+
| [공지] [자료실] [계약/증빙] [회의록] [인허가] [내부문서]                       |
+--------------------------------------------------------------------------------+
| 제목 | 유형 | 공개범위 | 상태 | 게시기간 | 작성자 | 최근수정 | 작업             |
+--------------------------------------------------------------------------------+
```

### 핵심 UX

- 공개 콘텐츠와 내부 문서를 명확히 분리한다.
- 공개 범위는 전체, 조합원, 임원, 내부관리자로 나눈다.
- 문서는 조합원 상세, 총회 결과, 납부 증빙과 연결될 수 있어야 한다.

## 9. 연동상태

5개 앱 API 연동의 운영 상태를 보여주는 화면이다.

```text
+--------------------------------------------------------------------------------+
| 연동상태                                                     [전체 동기화]      |
| 외부 모듈 연결, 마지막 동기화, 오류를 확인합니다.                               |
+--------------------------------------------------------------------------------+
| 앱         | 모듈명       | 상태 | 마지막 동기화 | 읽기 | 쓰기 | 오류 | 작업        |
| VoteCast   | 총회관리     | 정상 | 5분 전        | 가능 | 보류 | 0건  | 상세        |

```

### 핵심 UX

- 1차 MVP는 읽기 연동을 기본값으로 표시한다.
- 쓰기 연동은 보류 또는 제한 상태로 둔다.
- 오류는 원인, 영향 범위, 재시도 버튼을 함께 제공한다.
- 외부 앱으로 이동하는 링크와 API 설정 상태를 제공한다.

## 모바일 대응

ERP 관리자 화면은 desktop 우선이다. 모바일은 조합원 포털과 간단한 승인/확인 업무 중심으로 대응한다.

- 사이드바는 drawer로 전환
- 테이블은 카드 리스트 또는 핵심 컬럼만 표시
- 대량 편집, 엑셀 내보내기, 복잡한 필터는 desktop 권장
- 총회 현장 체크인처럼 모바일성이 필요한 업무는 VoteCast 같은 전문 모듈에서 처리

## 1차 구현 추천 순서

1. 앱 쉘: 사이드바, 상단 바, 페이지 헤더
2. 대시보드: KPI, 주의 필요, 연동 상태, 모듈 요약
3. 조합원 목록: 검색, 필터, 테이블
4. 조합원 상세: 탭 구조와 요약 카드
5. 회계/자금 요약: KPI, 예산 집행, 지출 승인 대기
6. 연동상태: 5개 앱 연결 카드
7. 총회/토지/수지분석 요약 화면
8. 문서/공지 관리

## 열린 결정 사항

구현 전에 다음 사항은 한 번 더 확인한다.

- ERP를 새 저장소로 만들지, 현재 `dbapt-erp` 폴더를 Git 저장소로 초기화할지
- Next.js App Router 기준으로 새 프로젝트를 만들지
- shadcn/ui 설치와 테마 토큰을 어느 시점에 적용할지
- Supabase는 ERP용 새 프로젝트를 쓸지, 기존 앱 DB를 읽기 연동할지
- 1차 MVP 데이터를 mock으로 시작할지, peopleON/db-landon부터 실제 API를 붙일지
